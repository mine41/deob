# PSDissect: A CFG-Guided, Semantics-Preserving Interactive Deobfuscation Framework for PowerShell Scripts

PSDissect is a research-oriented PowerShell deobfuscation toolkit for analyzing obfuscated and dynamically generated scripts. It combines CFG-guided rebuilding, interactive debugging, and round-level replay to recover readable code while preserving executable semantics as much as possible.

For normal use, you only need to remember these 3 entry points:

| Script | Purpose |
| --- | --- |
| `Rebuild-Deobfuscated.ps1` | Automatic multi-round deobfuscation that produces the final `.rebuilt.ps1`. This is the recommended main entry point. |
| `Debug-Deobfuscation.Wpf.ps1` | Step-by-step debugging with CFG view, variable stack, dynamic subgraphs, and live rebuild preview. |
| `Review-RoundReplay.Wpf.ps1` | Replay one round generated by `Rebuild` for investigation and troubleshooting. |

## Requirements

- The host matters. Run the tool with the same host family as the target script.
  - Scripts intended for Windows PowerShell 5.1: use `powershell.exe`
  - Scripts intended for PowerShell 7+: use `pwsh`
- `Debug-Deobfuscation.Wpf.ps1` and `Review-RoundReplay.Wpf.ps1` depend on WPF and only run on Windows.
- Installing Graphviz `dot` is recommended.
  - If installed: PNG graphs can be generated and shown in the UI
  - If not installed: core analysis still works, but graph visualization is limited

## Quick Start

```powershell
# 1) Automatic multi-round deobfuscation
.\Rebuild-Deobfuscated.ps1 -ScriptPath .\in\in.ps1

# 2) Debug mode
.\Debug-Deobfuscation.Wpf.ps1 -ScriptPath .\in\in.ps1

# 3) Replay a round
.\Review-RoundReplay.Wpf.ps1 -WorkDir .\in\in.rebuilt.ps1.work -Round 1
```

If you prefer launching the host explicitly:

```powershell
pwsh -NoProfile -File .\Rebuild-Deobfuscated.ps1 -ScriptPath .\in\in.ps1
pwsh -NoProfile -Sta -File .\Debug-Deobfuscation.Wpf.ps1 -ScriptPath .\in\in.ps1
pwsh -NoProfile -Sta -File .\Review-RoundReplay.Wpf.ps1 -WorkDir .\in\in.rebuilt.ps1.work -Round 1
```

For `powershell.exe -File`, prefer plain boolean-compatible values for entry parameters:

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\Rebuild-Deobfuscated.ps1 -ScriptPath .\in\in.ps1 -FullOutput false -SafeMode true
```

Supported boolean forms are `true/false`, `$true/$false`, `1/0`, `yes/no`, and `on/off`.

## Typical Workflow

### 1. Automatic Deobfuscation

Start with:

```powershell
.\Rebuild-Deobfuscated.ps1 -ScriptPath .\target.ps1
```

Output:

- Rebuilt script: `target.rebuilt.ps1`
- Work directory: `target.rebuilt.ps1.work` when `-FullOutput $true`

The work directory usually contains:

- `roundNN.in.ps1`
- `roundNN.out.ps1`
- `roundNN.execution.log`
- `roundNN.report.json`
- `roundNN.cfg.dot`
- `roundNN.cfg.png` if `dot` is installed

Best for:

- Getting a usable result quickly
- Recursive multi-round peeling
- Producing round data for later replay

### 2. Debug Mode

```powershell
.\Debug-Deobfuscation.Wpf.ps1 -ScriptPath .\target.ps1
```

Features:

- Prompts for UI language at startup
- Supports step, `Run All`, and reset
- Lets you inspect and modify variables
- Shows the current CFG node with a red highlight box
- Adds dynamic subgraphs into the current graph view
- Can export the current rebuilt result

Default work directory:

- `target.ps1.debug.work`

Common files:

- `debug.execution.log`
- `debug.cfg.dot`
- `debug.cfg.png`
- `debug.ui-error.log`

After export, it also writes:

- `debug.out.ps1`
- `debug.report.json`

Best for:

- Seeing exactly where execution is
- Supplying missing variables manually
- Confirming when dynamic code is generated and when control enters a subgraph

### 3. Replay a Round

```powershell
.\Review-RoundReplay.Wpf.ps1 -WorkDir .\target.rebuilt.ps1.work -Round 1
```

Features:

- Prompts for UI language at startup
- Replays round files produced by `Rebuild`
- Supports step backward, step forward, reset, and run to end
- Shows node data, variable state, and applied or skipped replacements for that round

Notes:

- `Review-RoundReplay.Wpf.ps1` reads a `*.work` directory produced by `Rebuild-Deobfuscated.ps1`
- It is not intended to read `debug.work`

## Parameter Reference

### `Rebuild-Deobfuscated.ps1`

```powershell
.\Rebuild-Deobfuscated.ps1 -ScriptPath .\target.ps1 [options]
```

| Parameter | Default | Description |
| --- | --- | --- |
| `-ScriptPath` | required | Input script |
| `-OutPath` | `<ScriptPath>.rebuilt.ps1` | Final rebuilt script |
| `-WorkDir` | `<OutPath>.work` | Round output directory, used only when `FullOutput=$true` |
| `-FullOutput` | `$true` | Whether to keep per-round artifacts |
| `-OverlapStrategy` | `Inner` | Overlap resolution strategy: `Inner` / `Outer` |
| `-VariableConflictPolicy` | `skip` | Multi-value variable policy: `skip` / `last` |
| `-MaxRounds` | `10` | Maximum number of rounds |
| `-MaxIterations` | `1000` | Maximum iterations per round |
| `-MaxTotalNodes` | `50000` | Maximum node visits per round |
| `-GlobalTimeBudgetMs` | `120000` | Overall time budget in milliseconds. `0` means unlimited. |
| `-DynamicTimeBudgetMs` | `15000` | Time budget for one dynamic expansion in milliseconds. `0` means unlimited. |
| `-SafeMode` | `$true` | Whether to keep safety protections enabled |
| `-PreExecutionGateMode` | `Balanced` | Complexity-aware pre-execution gate: `Disabled` / `Conservative` / `Balanced` / `Aggressive` |
| `-OptimizationProfile` | `Default` | Behavior-tuning profile: `Default` / `Cmdline` / `AdaptiveCoverage` |
| `-RunMetadataPath` | unset | Optional JSON summary output path for run metadata |
| `-DryRun` | `$false` | Analyze only, do not write the final output |

Common examples:

```powershell
# Fast mode: final rebuilt script only, no round artifacts
.\Rebuild-Deobfuscated.ps1 -ScriptPath .\in\in.ps1 -FullOutput $false

# Use the last observed value when variable readings conflict
.\Rebuild-Deobfuscated.ps1 -ScriptPath .\in\in.ps1 -VariableConflictPolicy last

# Analysis only, no final output file
.\Rebuild-Deobfuscated.ps1 -ScriptPath .\in\in.ps1 -DryRun

# Also write a compact run summary JSON
.\Rebuild-Deobfuscated.ps1 -ScriptPath .\in\in.ps1 -RunMetadataPath .\run-meta.json
```

Optimization profiles:

- `Default`: standard behavior.
- `Cmdline`: tuned for command-line style samples and more conservative about some rewrites.
- `AdaptiveCoverage`: prioritizes finishing within budget while preserving one more chance for high-value follow-up recovery after lightweight peeling.

Pre-execution gate modes:

- `Disabled`: do not apply the gate. The tool behaves like the old always-deepen strategy.
- `Conservative`: stop or downscale only obviously risky or very expensive fragments.
- `Balanced`: default mode. Good general setting for fixed-budget experiments such as `120s`.
- `Aggressive`: prioritize finishing within budget. More fragments are shallow-processed or skipped early.

The gate can classify a fragment as:

- `Full`: continue normal analysis and execution
- `Shallow`: continue, but tighten budgets and skip the most expensive stages
- `Stop`: do not deepen further; keep the current recovered text or evidence instead

Recommended baseline for a `120s` experiment:

```powershell
.\Rebuild-Deobfuscated.ps1 -ScriptPath .\target.ps1 -FullOutput $false -MaxRounds 5 -GlobalTimeBudgetMs 120000 -DynamicTimeBudgetMs 5000 -SafeMode $false -PreExecutionGateMode Balanced
```

Round reports may also include gate-related fields such as `GateMode`, `GateDecision`, `GateScore`, `GateReasons`, and `GateMetrics`.

### `Debug-Deobfuscation.Wpf.ps1`

```powershell
.\Debug-Deobfuscation.Wpf.ps1 -ScriptPath .\target.ps1 [options]
```

| Parameter | Default | Description |
| --- | --- | --- |
| `-ScriptPath` | required | Target script |
| `-WorkDir` | `<ScriptPath>.debug.work` | Debug work directory |
| `-OverlapStrategy` | `Inner` | Overlap strategy used by preview export |
| `-MaxIterations` | `1000` | Maximum iterations |
| `-MaxTotalNodes` | `50000` | Maximum node visits |
| `-DynamicTimeBudgetMs` | `60000` | Dynamic expansion budget in milliseconds |
| `-NoUI` | off | Do not open the window. Print an initialization summary only. |

Examples:

```powershell
.\Debug-Deobfuscation.Wpf.ps1 -ScriptPath .\in\in.ps1
.\Debug-Deobfuscation.Wpf.ps1 -ScriptPath .\in\in.ps1 -NoUI
```

### `Review-RoundReplay.Wpf.ps1`

```powershell
.\Review-RoundReplay.Wpf.ps1 [options]
```

| Parameter | Default | Description |
| --- | --- | --- |
| `-WorkDir` | chosen in UI | `*.work` directory produced by `Rebuild` |
| `-Round` | chosen in UI | Round to replay. In `-NoUI` mode, the latest round is used by default. |
| `-CheckpointInterval` | `200` | Variable-state checkpoint interval |
| `-NoUI` | off | Do not open the window. Print a summary only. |

Examples:

```powershell
.\Review-RoundReplay.Wpf.ps1 -WorkDir .\in\in.rebuilt.ps1.work -Round 1
.\Review-RoundReplay.Wpf.ps1 -WorkDir .\in\in.rebuilt.ps1.work -NoUI
```

## Usage Notes

- The usual order is: `Rebuild` first, then `Debug` if needed, then `Replay` when you need round-level investigation
- If the target script depends on a specific PowerShell version, use the same host version for the tool
- UI scripts should ideally be started with `-Sta`. If not, they will try to restart themselves in the same host with STA enabled
- In debug mode, manually assigned variable values are interpreted as PowerShell expressions, for example:

```powershell
123
'abc'
$null
''
```

## Boolean Parameter Caveat

For boolean parameters such as `-FullOutput` and `-SafeMode`, the most reliable way is to run the script directly inside the current PowerShell session:

```powershell
.\Rebuild-Deobfuscated.ps1 -ScriptPath .\in\in.ps1 -FullOutput $false
```

Notes:

- `pwsh -File ... -FullOutput:$false` works correctly
- `powershell.exe -File ... -FullOutput:$false` on Windows PowerShell 5.1 may treat `$false` as a string and fail parameter binding

## Data and Evaluation Artifacts

The `DATA` directory contains the small manual-check benchmark, CFG examples, large-scale experiment outputs, and the scripts used to reproduce the paper's evaluation tables.

Top-level folders:

- `DATA\case100`: 100-sample benchmark used for manual recovery-accuracy checking.
  - `obfuscated-Dataset`: obfuscated inputs.
  - `originals`: matching reference scripts used for manual intent checks.
  - `results-psdissect`: PSDissect outputs for the 100 samples.
  - `results-powerpeeler`: PowerPeeler outputs for the same samples.
  - `results-ivd`: IVD outputs for the same samples.
  - `summary.md`: manual inspection summary for Table 2, including per-sample validity/correctness notes.
- `DATA\cfg-showcase`: small source examples for CFG visualization.
  - `01-if-elseif-else.ps1` through `06-iex.ps1`: control-flow examples.
  - `Generate-Showcase-CFGs.ps1`: helper that generates DOT/PNG CFG outputs.
  - `out`: generated showcase CFG artifacts.
- `DATA\D-Script & D-Cmdline Results`: large-scale D-Script and D-Cmdline experiment outputs.
  - `ExecutionSummary`: execution inventories, validity records, and timing summaries used for efficiency calculations.
  - `SensitiveDataDetect`: sensitive-data detection outputs generated by `Detect-SensitiveData.ps1`.
  - `Post-Deobfuscation Evaluation`: semantic-consistency and residual-obfuscation evaluation outputs.
- `DATA\evaluation scripts`: offline evaluation scripts used to compute the paper tables from collected artifacts.

Evaluation scripts:

- `Detect-SensitiveData.ps1`: scans a dataset/output directory for recovered sensitive indicators such as IPs, URLs, file paths, registry keys, and related path-like artifacts. It is used for Table 3, "Sensitive Data Recovery"; the generated results are stored under `DATA\D-Script & D-Cmdline Results\SensitiveDataDetect`.
- `Compare-ApiSemanticConsistency.ps1`: compares baseline and candidate `effectiveApiSequence` data from existing `results.jsonl` or `results.json` files, optionally restricted by an inventory of valid samples. It is used for Table 4, "Semantic Consistency"; the generated results are stored under `DATA\D-Script & D-Cmdline Results\Post-Deobfuscation Evaluation`.
- `Compare-CollectedObfuscationResidual.ps1`: compares residual obfuscation-burden features from collected `results.json` files, using validity inventories and weighted residual metrics. It is used for Table 5, "Obfuscation Burden Reduction"; the generated results are stored under `DATA\D-Script & D-Cmdline Results\Post-Deobfuscation Evaluation`.
- `Test-DeobValidity.ps1`: lightweight helper that checks whether an output script is syntactically valid, non-empty, and not identical to the input after whitespace removal.

Paper table mapping:

| Paper Table | Metric | Method / Script | Result Location |
| --- | --- | --- | --- |
| Table 2 | Recovery Accuracy | Manual inspection against `DATA\case100\originals` | `DATA\case100\summary.md` |
| Table 3 | Sensitive Data Recovery | `DATA\evaluation scripts\Detect-SensitiveData.ps1` | `DATA\D-Script & D-Cmdline Results\SensitiveDataDetect` |
| Table 4 | Semantic Consistency | `DATA\evaluation scripts\Compare-ApiSemanticConsistency.ps1` | `DATA\D-Script & D-Cmdline Results\Post-Deobfuscation Evaluation` |
| Table 5 | Obfuscation Burden Reduction | `DATA\evaluation scripts\Compare-CollectedObfuscationResidual.ps1` | `DATA\D-Script & D-Cmdline Results\Post-Deobfuscation Evaluation` |
| Table 6 | Efficiency | Timing summaries from execution runs | `DATA\D-Script & D-Cmdline Results\ExecutionSummary` |

## File Overview

- `Rebuild-Deobfuscated.ps1`: main automatic multi-round deobfuscation entry point
- `Debug-Deobfuscation.Wpf.ps1`: interactive debugger
- `Review-RoundReplay.Wpf.ps1`: round replay UI
- `Generate-CFG.ps1`: CFG generation and DOT export
- `Execute-CFG.ps1`: execution engine and debug session API
- `Ui-Localization.ps1`: bilingual UI text for debug and replay
- `ast.ps1`: standalone AST helper for parsing PowerShell input through `Get-Ast`
- `ConvertTo-Expression-origin.ps1`: serialization helper dot-sourced by `Execute-CFG.ps1`
